# Smart Notes — 智能笔记生成器

输入链接，自动提取内容并生成结构化学习笔记。

## 工作流程

### 第一步：运行提取脚本

用 `--no-ai` 模式运行提取脚本，只提取原始内容，不调用外部 AI：

```bash
./smart-notes/run.sh "用户提供的链接" --no-ai
```

脚本会自动识别平台（抖音/微信），提取内容到 `output/内容标题/raw/` 目录。

首次运行会自动安装依赖（playwright、whisper 等），可能需要几分钟。

### 第二步：读取提取的内容

脚本运行完后，读取 raw 目录下的文件：

- `raw/transcript.txt` — 抖音视频的语音转文字
- `raw/content.md` — 微信文章的正文
- `raw/comments.txt` — 评论（如果有）

同时检查 `supplements/extra.md` 是否有用户添加的补充信息。

### 第三步：生成笔记

根据读取的内容，按下面的格式生成笔记，写入 `output/` 目录。

## 笔记格式

```markdown
# 标题

## 来源
- 链接：（原始链接）
- 作者：（如果有）
- 发布时间：（如果有）
- 标签：（如果有）
- 数据：X 赞 · X 评论 · X 收藏 · X 转发（如果有）

## 摘要
2-3 句话概括核心观点，不超过 100 字。直接写观点，不要用"该视频""该文章"开头。

## 要点
- 3-8 条核心要点
- 每条不超过 30 字
- 直接写结论

## 正文
用小标题分段，直接写知识点。评论区有价值的信息融入正文。

## 评论
保留有价值的讨论，忽略纯求资源、纯表情的。标记为（作者）的是原作者的回复。
如果没有评论内容则省略此章节。
```

## 写作规则

- 直接写知识点，不要转述（不说"作者指出""视频中提到"）
- 禁止用词：该内容、该方法、该视频、该文章、该技巧、作者指出、视频讲解、文章介绍、本视频、本文
- 口语化但不随意，像有经验的同事在讲东西
- 用短句，能一句说清的不用两句

## 输出位置

笔记写入 `output/内容标题/output/` 目录，文件名用版本号：
- 首次生成：`v1.md`
- 如果已有 v1，则生成 `v2.md`，以此类推

## 注意事项

- 如果脚本提示需要安装 Python 或 ffmpeg，告知用户运行 `brew install python@3.12` 和 `brew install ffmpeg`
- 抖音评论获取需要登录，脚本会交互式询问，在 `--no-ai` 模式下可能跳过。如果用户需要评论，建议手动运行 `./smart-notes/run.sh "链接" --no-ai` 后再让 Claude 生成笔记
- 提取过程中如果遇到网络问题，可以重试
