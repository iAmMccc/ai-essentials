# UI Diff — UI 走查助手

对比 UI 设计稿与 App 截图，自动检查间距、对齐、颜色、字号等视觉差异。

基于 AI 视觉能力，替代人工拉线对比，适用于 Claude Code、Cursor 等 AI 编辑器。

## 安装

新建一个文件夹，在里面执行：

```bash
curl -sL https://raw.githubusercontent.com/iAmMccc/ai-essentials/main/install-skill.sh | bash -s ui-diff
```

安装后的目录结构：

```
ui-review/
├── CLAUDE.md           ← Claude Code 自动读取
├── AGENTS.md           ← Cursor 自动读取
└── images/             ← 自动创建，放对比图片
```

不依赖 `.claude` 或 `.cursor` 隐藏目录，Claude Code 和 Cursor 都能直接使用。

## 使用

### 1. 批量对比（推荐）

在 `images/` 下按页面建子文件夹，每个子文件夹放 2 张图：

```
images/
├── 确认订单/
│   ├── 1.png         ← 设计稿
│   └── 2.png         ← App 截图
├── 全部功能/
│   ├── design.png
│   └── screenshot.png
```

打开 Claude Code 或 Cursor，执行：

```
/ui-diff images
```

对比完成后，每个子文件夹会生成 `diff.md` 报告：

```
images/
├── 确认订单/
│   ├── 1.png
│   ├── 2.png
│   └── diff.md       ← 自动生成
```

图片不需要特定命名，AI 会自动识别哪张是设计稿、哪张是截图：

| 优先级 | 规则 | 示例 |
|--------|------|------|
| 1 | 文件名为数字，编号小的是设计稿 | `1.png` = 设计稿，`2.png` = 截图 |
| 2 | 状态栏时间 9:41 是设计稿 | Apple 设计模板默认时间 |
| 3 | 有 debug 悬浮按钮的是截图 | 齿轮、SIT/Flutter、甲壳虫图标 |

### 2. 单组对比

```
/ui-diff 设计稿.png 截图.png
```

### 3. 直接贴图

在对话中粘贴两张图片，说明哪个是设计稿即可。

## 对比维度

- **布局与间距** — padding、margin、元素间距
- **对齐** — 水平/垂直对齐关系
- **尺寸** — 元素宽高、图标大小
- **文字** — 字号、字重、行高、颜色、内容
- **颜色与样式** — 背景色、边框、圆角、阴影
- **元素增减** — 设计稿有但截图缺失，或截图多出的元素

只报告有差异的项，没问题的不会列出。

## 输出示例

```markdown
## 确认订单

> 设计稿：1.png | 截图：2.png

### 差异（3 处）

#### 1. 「导出数量」缺少补充说明文字
- **位置：**导出数量行下方
- **设计稿：**有灰色小字「单次最高5000条...」
- **实际截图：**没有补充说明
- **建议修改：**补上灰色提示文字

#### 2. 底部按钮样式不一致
- **位置：**页面底部固定按钮
- **设计稿：**左侧「应付总额 ¥500」+ 右侧黄色按钮「立即支付」
- **实际截图：**蓝色按钮「提交订单」，无金额
- **建议修改：**确认是否为同一业务场景
```

## 注意事项

- 间距数值基于视觉估算（标注为「约 Xpx」），不是像素级精确测量
- 截图中的 debug 悬浮按钮、测试标注（红框/编号）会被自动忽略
- 设计稿和截图分辨率不同时，会在报告中说明
