# UI Diff — UI 走查助手

对比 UI 设计稿与 App 截图，自动检查间距、对齐、颜色、字号等视觉差异。

基于 AI 视觉能力，替代人工拉线对比，适用于 Claude Code、Cursor 等 AI 编辑器。

## 安装

### 方式一：安装到项目（推荐）

在项目根目录执行：

```bash
curl -sL https://raw.githubusercontent.com/iAmMccc/ai-essentials/main/install-skill.sh | bash -s ui-diff
```

自动安装到 `.claude/skills/` 和 `.cursor/skills/`，Claude Code 和 Cursor 都能识别。

### 方式二：直接在 Claude 中使用

将 `SKILL.md` 内容粘贴到 Claude 对话中，或手动复制到：

- Claude Code：`.claude/skills/ui-diff/SKILL.md`
- Cursor：`.cursor/skills/ui-diff/SKILL.md`

## 使用

### 1. 批量对比（推荐）

在项目下按页面建子文件夹，每个子文件夹放 2 张图：

```
pages/
├── 确认订单/
│   ├── 1.png         ← 设计稿
│   └── 2.png         ← App 截图
├── 全部功能/
│   ├── design.png
│   └── screenshot.png
```

打开 Claude Code 或 Cursor，执行：

```
/ui-diff pages
```

对比完成后，每个子文件夹会生成 `diff.md` 报告：

```
pages/
├── 确认订单/
│   ├── 1.png
│   ├── 2.png
│   └── diff.md       ← 自动生成
```

图片不需要特定命名，AI 会自动识别哪张是设计稿、哪张是截图：

| 优先级 | 规则 | 示例 |
|--------|------|------|
| 1 | 文件名为数字，编号小的是设计稿 | `1.png` = 设计稿，`2.png` = 截图 |
| 2 | 状态栏时间 9:41 是设计稿 | Apple 设计模板默认时间 |
| 3 | 有 debug 悬浮按钮的是截图 | 齿轮、SIT/Flutter、甲壳虫图标 |

### 2. 单组对比

```
/ui-diff 设计稿.png 截图.png
```

### 3. 直接贴图

在对话中粘贴两张图片，说明哪个是设计稿即可。

## 对比策略

采用三步法提升检查精确度：

1. **建立参照体系** — 用 iOS 系统组件（导航栏 44pt、TabBar 49pt 等）作为标尺估算间距
2. **分区逐元素扫描** — 将屏幕分为 3-4 个区域，逐个元素检查（含分隔线有无、组件内部间距等）
3. **标注置信度** — 每条差异标注确定/高度疑似/待确认，宁可多报不可漏报

## 对比维度

- **布局与间距** — padding、margin、元素间距
- **对齐** — 水平/垂直对齐关系
- **尺寸** — 元素宽高、图标大小
- **文字** — 字号、字重、行高、颜色、内容
- **颜色与样式** — 背景色、边框、圆角、阴影
- **元素增减** — 设计稿有但截图缺失，或截图多出的元素

只报告有差异的项，没问题的不会列出。

```markdown
## 差异汇总

共计 2 条差异项，其中功能差异 1 条，UI 细节差异 1 条。

## 功能差异

### 1. 「导出数量」缺少补充说明文字 `确定`
- **位置：**导出数量行下方
- **设计稿：**有灰色小字「单次最高5000条...」
- **实际截图：**没有补充说明
- **建议修改：**补上灰色提示文字

## UI 细节差异

### 2. 标题与副标题间距偏大 `高度疑似`
- **位置：**页面标题下方
- **设计稿：**间距约 4pt
- **实际截图：**间距约 8pt，参照导航栏高度估算
- **建议修改：**标题下间距减半
```

## 注意事项

- 间距数值基于参照物估算（标注为「约 Xpt」），不是像素级精确测量
- 截图中的 debug 悬浮按钮、测试标注（红框/编号）会被自动忽略
- 设计稿和截图分辨率不同时，会在报告中说明
